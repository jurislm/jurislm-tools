將今日工作總結寫入 Notion 每日日誌 DB，並同步更新 MEMORY.md 中的重要資訊。

## 使用方式

```
/notion-workflow:daily-log
/notion-workflow:daily-log 今天完成了 JurisLM API 設計，明天要開始前端
```

若未提供內容，主動詢問用戶：
1. 今日完成了什麼？
2. 明日計畫？
3. 任何需要記在 MEMORY.md 的重要資訊？

## 步驟一：收集今日資訊

若用戶已提供內容，直接解析；否則依序詢問：

**必填：**
- 今日完成事項（可以條列式輸入）

**選填（可跳過）：**
- 明日計畫
- 心情/能量狀態（高能/普通/疲憊）
- 備註、反思

**若習慣尚未記錄，也一併詢問：**
- 今日運動/閱讀/冥想完成了嗎？

## 步驟二：寫入每日日誌 DB

查詢今日日誌是否已存在（DB ID：`ed2cb0ab1565439e8bc51b521bf63618`）：

**若已存在** → 更新現有頁面（補充或覆蓋內容）
**若不存在** → 建立新頁面

寫入欄位：
- 日期（Title）：`YYYY-MM-DD`
- 今日完成：用戶提供的完成事項
- 明日計畫：用戶提供的計畫
- 心情/能量：Select 值
- 習慣欄位（若用戶提供）
- 備註

## 步驟三：萃取「重要洞察」與「待跟進」

**每次寫入或更新日誌後，必須執行此步驟。** 排程任務（如 `daily-learning-log`）執行時同樣必做。

### 重要洞察（必填）

從當天的「今日內容」中萃取 1-3 條最有價值的洞察，寫入「重要洞察」欄位。

**萃取標準（符合任一即可）：**
- 反直覺或顛覆既有認知的觀點
- 可應用於投資決策、產品開發、或個人成長的具體框架
- 來自專家/業界的一手資訊或獨特見解
- 跨領域可遷移的思考模型

**格式：** 每條洞察一行，精簡到 1-2 句話，點出「為什麼這很重要」。不加 bullet point 符號，直接用換行分隔。

**範例：**
```
台積電先進封裝外溢 → 封測股從後段加工升格為 AI 晶片關鍵價值鏈（產業隊長）
漲價要區分「每次都漲且漲很久」vs「久久漲一次」，決定追不追（產業隊長）
不配息 ETF 破 640 億，高資產族群從「領現金」轉向「複利效率＋節稅」（柴鼠兄弟）
```

### 待跟進（有則填寫）

從當天內容中提取需要後續追蹤的事項：
- 需要驗證的判斷或預測
- 需要持續觀察的股票/產業動向
- 需要在特定時間點回頭檢視的決策
- 待完成的任務或行動

**格式：** 每條一行，標註追蹤原因。

**範例：**
```
追蹤力成（6239）季線整理後是否站穩 → 驗證「挫折只是過程」判斷
觀察被動元件漲價是否僅為一日行情 → 華新科/國巨後續走勢
```

### 寫入方式

使用 `notion-update-page` 更新當天日誌頁面的 `重要洞察` 和 `待跟進` 欄位。
- 若欄位已有內容，**追加**新內容（換行後接續），不覆蓋既有內容
- 若當天內容確實沒有值得記錄的洞察，可留空並跳過

## 步驟四：精簡收斂 MEMORY.md

**每次寫完日誌後，固定執行此步驟。**

先使用 `notion-fetch` 讀取現有 MEMORY.md（頁面 ID：`3264f5ac5c2b812d9990eddaab24169e`），再依以下邏輯進行更新：

### 必定收斂（每日執行）
- 將今日完成事項萃取為 1–3 條精簡重點，更新至 MEMORY.md 的「近期進展」區塊
- 若明日計畫存在，更新「下一步」區塊

### 條件性更新（有則補充）
- 重大決策（例：「決定使用 X 技術方案」）→ 更新「重要決策」區塊
- 專案里程碑（例：「完成 API v1 設計」）→ 更新對應專案區塊
- 新的待追蹤事項（例：「需要聯繫 Brian」）→ 更新「待追蹤」區塊

### 精簡原則
- 同類資訊合併，不堆疊重複條目
- 過時的「近期進展」或「下一步」直接覆蓋，不保留歷史
- 保持 MEMORY.md 精簡，每個區塊以 3–5 條為上限

使用 `notion-update-page` 更新 MEMORY.md：
- 頁面 ID：`3264f5ac5c2b812d9990eddaab24169e`
- 更新「最後更新日期」欄位

## 步驟五：回報

## 注意事項

- 若用戶已在本次對話中提過相關資訊，自動帶入，不重複詢問
- MEMORY.md 只記錄跨對話需要保留的資訊，不記錄瑣碎細節
- 所有輸出使用繁體中文
